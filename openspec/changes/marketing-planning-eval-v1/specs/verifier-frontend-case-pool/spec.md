## ADDED Requirements

### Requirement: Case-pool persistence remains lightweight for large project outputs
The verifier frontend case-pool behavior SHALL persist durable case source data but SHALL NOT persist oversized transient run artifacts such as full traces, raw SSE streams, full card payloads, judge raw model output, or complete frontend view objects in browser session storage.

#### Scenario: Large SSE run completes
- **WHEN** a batch case produces a large SSE or card payload
- **THEN** the frontend keeps the current in-memory run available for display but persists only the lightweight case source fields needed to rerun or reload the pool.

#### Scenario: Browser storage quota is reached
- **WHEN** browser storage rejects a case-pool save
- **THEN** batch polling and completed case display continue, and unrelated cases are not aborted.

### Requirement: Output and reference are visually aligned
The verifier frontend case-pool behavior SHALL render output and reference with the same top-level shape and comparable visual treatment when a project supplies structured references.

#### Scenario: Structured output and reference
- **WHEN** a case has JSON-like output and reference summaries
- **THEN** the table and details views render formatted JSON with matching cell sizing rather than lossy comma-joined text.

#### Scenario: Reference generated by judge
- **WHEN** the input has no reference and judge generates one
- **THEN** the frontend marks the reference source as judge-generated and displays it in the same comparable panel as provided references.

### Requirement: Batch frontend uses unified APIs
The verifier frontend case-pool behavior SHALL submit generated, uploaded, saved, and manually edited cases through the unified batch job APIs.

#### Scenario: Marketing-planning batch starts
- **WHEN** the user runs marketing-planning cases from the summary page
- **THEN** the page calls the generic batch API and renders per-case status, judge summary, attribution summary, cluster, and check results from protocol outputs.
