## Project-specific placement decisions

- QA provided input/output/reference, contexts, scenario, multidimensional scoring, and QA taxonomy belong in project adapter/configuration extensions and project standards, not hardcoded generic core fields.
- QA uploaded datasets enter the generic case-pool and RunTrace protocols through JSON-compatible normalization; per-row isolation is a batch/core requirement.
- marketing-planning repository paths, startup behavior, SSE/API shape, and integration-risk constraints belong in project configuration and adapter hooks.
- marketing-planning-intent shares the marketing-planning service boundary but must declare its own endpoint, single-turn graph behavior, and judge boundary in project configuration/hooks.
- Generic core may only define extension points, state records, evidence records, gates, transitions, and persistence contracts needed by all projects.
